# Accessibility

## Contrast minimums
- meet WCAG 2.2 AA expectations for core text and controls
- validate all accent-based surfaces before release

## Dynamic type rules
- support text scaling without clipping actions
- allow cards and lists to reflow naturally

## Screen reader labels
- every interactive element needs an explicit accessible label
- map handoff actions must identify the destination app
- connected states must expose status and next action

## Keyboard navigation rules
- all web flows require logical tab order
- every focusable element needs a visible focus state
- planner and auth flows must remain fully keyboard operable

## Touch target size
- minimum 44x44 target size or equivalent

## Color reliance warnings
- do not use color alone for errors, confidence, or connection state
- pair color with iconography, labels, or helper text
