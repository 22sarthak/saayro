# MVP Scope

## In-scope screens
- marketing and landing pages
- auth entry and login choice screens
- onboarding and preference capture
- trip list and trip hub
- planner workspace and itinerary
- Buddy surfaces
- exports and sharing panels
- profile and settings
- connected travel overview with mock states

## In-scope flows
- create trip
- edit itinerary
- ask Buddy for guidance
- export or share a trip
- choose preferred maps app and hand off a route
- connect a travel account in mock mode

## Out-of-scope features
- real payments
- production auth backend wiring
- live Google or OTP auth completion
- inbox parsing and live calendar syncing
- full offline mode
- video understanding pipeline
- advanced model orchestration

## Launch constraints
- mock data must be clearly framed
- all primary empty, loading, error, and partial states must exist
- web and mobile terminology must stay aligned

## Acceptance criteria
- docs explain the shell clearly enough for implementation
- screen inventory and contracts do not conflict
