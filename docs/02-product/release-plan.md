# Release Plan

## Alpha
- internal shell review
- design and UX validation
- terminology cleanup

## Internal beta
- interactive shell with reusable components
- documented API contracts reviewed against likely implementation
- early QA on key flows

## Closed beta
- limited external testers
- feedback on trust, clarity, and usefulness
- monitor crash-free sessions and export success

## Public beta
- broader acquisition with beta messaging
- focus on highest-friction planning and Buddy moments

## Launch readiness checklist
- core journeys implemented consistently
- analytics events instrumented
- permissions and disclaimer copy approved
- QA matrix complete for supported platforms
