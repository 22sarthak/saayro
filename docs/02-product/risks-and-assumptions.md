# Risks And Assumptions

## Assumptions
- users value portability and clarity over lock-in
- travel trust cues are more important than maximum feature breadth
- India-specific travel variability requires strong uncertainty communication

## Risks
- data freshness can drift from reality
- trust can break if Buddy overclaims
- product complexity can overwhelm the shell
- integrations can be fragile
- connected features create privacy concerns
- scope creep can blur the build order

## Mitigations
- ship shell-first with explicit non-goals
- use confidence labels and disclaimers
- keep the contracts modular and explicit
