# Connected Travel Flow

## Connect account
1. user opens connected travel
2. app explains value, scope, and reversibility
3. user chooses a provider
4. mock connection completes with visible status

## Scan travel items
- app shows a scan or sync state
- imported items must be visibly marked as connected, not manually created

## Surface results
- group results by likely trip, transport, stay, or event type
- expose confidence labels where extraction quality may vary

## Attach to trip
1. user reviews a connected item
2. user attaches it to an existing or new trip
3. trip hub updates with the connected item state

## Add to calendar
- when supported later, allow calendar event creation or sync confirmation
- keep this clearly marked as future or placeholder in MVP

## Revoke permissions
- user can disconnect a provider from settings
- revoke flow must explain what data remains cached, if any
