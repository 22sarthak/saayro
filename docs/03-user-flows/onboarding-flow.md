# Onboarding Flow

## Welcome screens
- introduce Saayro as a premium travel companion
- explain Buddy as a guide, not generic chat
- reinforce portability and calm planning

## Optional preference capture
- travel style
- trip type
- preferred maps app
- interests such as food, pace, and comfort

## Permissions staging
- ask only when relevant
- defer notifications, mic, and camera until a feature entry point requires them
- explain the value and reversibility of each permission

## First trip prompt
- encourage users to create a first trip immediately
- allow destination-led, date-led, or idea-led entry

## Skipped onboarding behavior
- user can reach the trip shell without answering every question
- unset preferences should degrade gracefully and remain editable later
