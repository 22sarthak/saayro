# App Information Architecture

## Top-level sections
- marketing
- auth
- onboarding
- trips
- planner workspace
- Buddy
- exports and sharing
- connected travel
- profile and settings

## Screen grouping
- marketing and acquisition live primarily on web
- planning and Buddy exist on both web and mobile
- connected travel and settings are common but layout-adapted

## Relationship between web and mobile
- web supports deeper planning and multi-panel organization
- mobile supports in-trip access, quick updates, and fast handoff actions
- both must share the same trip model and terminology
