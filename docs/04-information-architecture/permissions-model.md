# Permissions Model

## Location
- request only when route relevance or nearby guidance requires it

## Camera
- request only at the moment of image capture

## Mic
- request only for voice capture entry points

## Photo library
- request only when the user imports an image

## Contacts later if ever added
- not in scope for MVP
- require separate value justification if introduced later

## Email access
- request only for connected travel and clearly scope access

## Calendar access
- request only when user wants trip events or sync support

## Notification permissions
- stage later in onboarding or after trip creation, not on first launch
