# Web App Spec

## Landing pages
- hero, positioning, feature explanation, and primary CTA
- editorial layout that expresses Ivory Atlas clearly

## Dashboard
- recent trips
- start new trip
- connected travel summary
- quick access to exports and Buddy

## Planner workspace
- multi-panel desktop layout
- itinerary editing with visible trip summary and action rail
- Buddy should be accessible without taking over the screen

## Auth pages
- Google and mobile login choices
- privacy and value framing kept concise

## Desktop layout behavior
- support wide screens with persistent context panels
- maintain readable max widths for long-form content
- preserve clear empty, loading, and error states
