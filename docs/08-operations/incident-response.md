# Incident Response

## Severity levels
- Sev 1: core app unavailable or critical user data risk
- Sev 2: major user flow degraded such as auth or exports
- Sev 3: partial feature issue with workaround
- Sev 4: low-impact defect or cosmetic issue

## Response owners
- engineering lead for technical containment
- product lead for user impact decisions
- support owner for user communication when applicable

## Response flow
- detect and classify
- contain impact
- communicate status
- resolve and verify
- log follow-up actions
