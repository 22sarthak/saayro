# Observability

## Logs
- structured logs for web, mobile, and backend where supported
- include request ids and trip-safe context

## Traces
- trace critical flows such as trip creation, export generation, and connected provider actions

## Front-end error monitoring
- capture uncaught UI exceptions and route-level failures
- tag by platform and release version

## API metrics
- request volume
- latency by endpoint
- error rates
- auth and export failure rates

## AI metrics later
- Buddy response latency
- safety issue counts
- helpfulness and correction signals
